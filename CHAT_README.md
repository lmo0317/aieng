# AI 음성 대화 시스템 구현 문서

## 개요

AIEng 프로젝트를 위한 음성 대화 기능이 구현되었습니다. Web Speech API를 활용하여 음성 인식과 음성 합성을 제공하며, 실시간 채팅 인터페이스를 통해 자연스러운 대화 경험을 제공합니다.

## 구현된 기능

### 1. 음성 입력 (Speech-to-Text)
- Web Speech API의 SpeechRecognition 활용
- 한국어 음성 인식 지원
- 실시간 음성-텍스트 변환
- 마이크 권한 요청 및 에러 처리
- 녹음 상태 시각적 피드백

### 2. 음성 출력 (Text-to-Speech)
- Web Speech API의 SpeechSynthesis 활용
- 한국어 보이스 자동 선택
- 자연스러운 음성 재생
- 재생 중지 기능
- 말하기 상태 시각적 피드백

### 3. 채팅 인터페이스
- 현대적이고 직관적인 UI 디자인
- 반응형 디자인 (모바일/태블릿/데스크톱)
- 플로팅 액션 버튼 (FAB)
- 모달 기반 채팅창
- 메시지 히스토리 표시
- 타이핑 인디케이터

### 4. 백엔드 연동
- Fetch API를 통한 /api/chat 엔드포인트 호출
- Server-Sent Events (SSE) 스트리밍 지원
- 실시간 AI 응답 표시
- 대화 컨텍스트 유지 (최근 10개 메시지)
- 에러 핸들링 및 재시도 로직

### 5. 접근성 (Accessibility)
- ARIA 라벨 및 역할 속성
- 키보드 네비게이션 지원 (ESC로 닫기, Enter로 전송)
- 포커스 관리
- 스크린 리더 최적화

## 사용 방법

### 1. 채팅 시작
1. 우측 하단의 보라색 마이크 버튼 클릭
2. 채팅 모달이 열립니다

### 2. 음성으로 대화
1. 입력창 좌측의 마이크 버튼 클릭
2. 마이크 권한 허용
3. 말하기 시작 (녹음 중 상태 표시)
4. 말을 마치면 자동으로 AI에게 전송
5. AI 응답을 음성으로 들음

### 3. 텍스트로 대화
1. 입력창에 메시지 입력
2. 전송 버튼 클릭 또는 Enter 키 입력
3. AI 응답을 텍스트와 음성으로 확인

### 4. 채팅 종료
1. 우측 상단의 X 버튼 클릭
2. 또는 ESC 키 입력
3. 모달 외부 영역 클릭

## 기술적 구현 상세

### Web Speech API 호환성

**음성 인식 (SpeechRecognition)**
- Chrome/Edge: 완전 지원 (webkitSpeechRecognition)
- Safari: 부분 지원 (iOS 14.5+)
- Firefox: 지원 안 함 (텍스트 입력만 가능)

**음성 합성 (SpeechSynthesis)**
- 모든 현대적 브라우저 지원
- 한국어 보이스 가용성은 브라우저/OS에 따라 다름

### API 연동

**엔드포인트:** `POST /api/chat`

**요청 형식:**
```json
{
  "message": "사용자 메시지",
  "history": [
    { "role": "user", "content": "이전 메시지" },
    { "role": "assistant", "content": "이전 응답" }
  ]
}
```

**응답 형식 (SSE 스트리밍):**
```
data: {"content": "응답", "done": false}
data: {"content": " 조각", "done": false}
data: {"content": "들", "done": false}
data: [DONE]
```

### 상태 관리

```javascript
const ChatState = {
    isModalOpen: false,      // 모달 열림 상태
    isRecording: false,      // 음성 녹음 중
    isSpeaking: false,       // 음성 재생 중
    isProcessing: false,     // AI 응답 대기 중
    messages: [],            // 대화 히스토리
    recognition: null,       // SpeechRecognition 인스턴스
    synthesis: null,         // SpeechSynthesis 인스턴스
    currentUtterance: null   // 현재 재생 중인 utterance
};
```

## UI/UX 특징

### 디자인 원칙
- **모던하고 깔끔한 디자인**: 그라데이션, 둥근 모서리, 부드러운 애니메이션
- **직관적인 인터랙션**: 명확한 버튼, 상태 표시, 피드백
- **반응형 레이아웃**: 모든 디바이스에서 최적의 경험

### 색상 시스템
- **주요 색상**: 보라색 그라데이션 (#667eea → #764ba2)
- **배경색**: 연한 회색 (#f7fafc)
- **텍스트**: 진한 회색 (#2d3744)
- **상태 표시**: 녹색 (녹음), 빨간색 (오류)

### 애니메이션
- 모달 슬라이드인 (0.3s)

- 버튼 호버 효과 (scale)
- 녹음 펄스 효과 (1s infinite)
- 타이핑 인디케이터 (1.4s infinite)

## 브라우저 호환성

| 브라우저 | 음성 인식 | 음성 합성 | 최소 버전 |
|---------|---------|---------|---------|
| Chrome | ✅ | ✅ | 25+ |
| Edge | ✅ | ✅ | 79+ |
| Safari | ⚠️ | ✅ | 14.5+ |
| Firefox | ❌ | ✅ | - |

**범례:**
- ✅ 완전 지원
- ⚠️ 부분 지원 (일부 기능 제한)
- ❌ 지원 안 함

## 에러 핸들링

### 음성 인식 오류
- `no-speech`: 음성이 감지되지 않음
- `audio-capture`: 마이크 접근 불가
- `not-allowed`: 마이크 권한 거부
- `network`: 네트워크 오류

### 백엔드 오류
- 네트워크 연결 실패
- API 서버 오류
- 타임아웃

### 대응 전략
- 사용자 친화적인 에러 메시지
- 자동 재시도 로직
- 그레이스풀 디그레이데이션 (음성 미지원 시 텍스트 입력)

## 보안 고려사항

1. **마이크 권한**: 사용자 명시적 허용 필요
2. **HTTPS**: 음성 인식은 HTTPS 환경에서만 작동
3. **데이터 전송**: 암호화된 HTTPS 연결 사용
4. **민감 정보**: 음성 데이터는 로컬에서만 처리 후 전송

## 성능 최적화

1. **메시지 히스토리**: 최근 10개만 유지하여 메모리 절약
2. **이벤트 디바운싱**: 불필요한 이벤트 처리 방지
3. **DOM 조작 최적화**: 최소한의 리플로우/리페인트
4. **스크롤 최적화**: requestAnimationFrame 사용

## 향후 개선 사항

1. **다국어 지원**: 영어, 일본어 등 추가 언어
2. **음성 커스텀**: 속도, 피치, 볼륨 조절
3. **대화 내보내기**: 텍스트 파일로 저장
4. **테마 설정**: 다크 모드 지원
5. **오프라인 모드**: PWA로 변환

## 문제 해결

### 음성 인식이 작동하지 않음
1. HTTPS 연결 확인
2. 마이크 권한 허용 확인
3. 브라우저 호환성 확인

### 음성 출력이 안 들림
1. 볼륨 확인
2. 브라우저 오디오 권한 확인
3. 한국어 보이스 설치 확인

### API 연결 실패
1. 서버 실행 상태 확인
2. CORS 설정 확인
3. 네트워크 연결 확인
