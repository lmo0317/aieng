# 🎬 뉴스 학습 콘텐츠 생성 에이전트 워크플로우

본 문서는 `/news [n]개 만들어줘` 명령 시 에이전트 팀이 어떻게 협업하여 데이터를 생성하는지 설명합니다.

## 1. 동작 원칙: "Scout Bulk, Produce Sequential"
데이터 유실을 방지하고 고품질의 해설을 보장하기 위해, 주제는 한꺼번에 수집(Scout)하되, 실제 콘텐츠 제작은 **하나씩 순차적으로(One by One)** 진행합니다.

---

## 2. 워크플로우 (텍스트 흐름도)

[시작: 사용자 요청 (/news 5개)]
      |
      v
[Step 0: 데이터 청소] ------------------> 오늘 날짜의 기존 뉴스 DB 삭제
      |
      v
[Step 1: 전체 주제 수집] ---------------> 엔터/스포츠/종합 뉴스 RSS 확보 (비율: 4:1)
      |
      +-----> [주제 1 가공 시작]
      |          (1) 콘텐츠 작가: 영어 문장 10개 생성
      |          (2) 1타 강사: 구문 분석 및 해설 작성
      |          (3) 기술 QA: JSON 검수 및 키워드 추출
      |          (4) 최종 감수관: 9대 기준 최종 승인
      |          (5) 서버 동기화: DB 저장 완료
      |
      +-----> [주제 2 가공 시작] (위 과정 반복)
      |
      +-----> [주제 3 가공 시작] (위 과정 반복)
      |
      +-----> [주제 4 가공 시작] (위 과정 반복)
      |
      +-----> [주제 5 가공 시작] (위 과정 반복)
      |
      v
[최종 완료 보고] -----------------------> "총 5개의 뉴스가 생성되었습니다."

---

## 3. 상세 단계별 역할

### **Step 0: 데이터 정제 (Cleanup)**
- `scripts/delete-today.js`를 실행하여 오늘 날짜로 저장된 기존 뉴스 데이터를 모두 삭제합니다. 
- 이는 데이터 중복을 방지하고 항상 최신 트렌드만 유지하기 위함입니다.

### **Step 1: 전체 수집 (Scout ALL)**
- 개선된 `scripts/fetch-rss.js`를 통해 **엔터테인먼트, 스포츠, 종합** 뉴스 리스트를 가져옵니다.
- 설정된 **우선순위(엔터 > 스포츠 > 종합)**와 **비율(5개 요청 시 4:1)**에 따라 에이전트가 오늘 다룰 주제를 확정합니다.

### **Step 2: 순차 가공 (Sequential Production)**
- **왜 하나씩 만드나요?**: 한 번의 응답에 5개 뉴스(문장 50개, 해설 50개)를 담으면 내용이 생략되거나 AI가 지치기 때문입니다.
- **작가 -> 강사 -> QA**가 팀을 이뤄 한 뉴스가 완벽해질 때까지 집중 마크합니다.

### **Step 3: 최종 승인 및 동기화 (Audit & Sync)**
- **감수관**이 "제목에 영어가 섞이지 않았는지", "한자가 없는지" 등을 최종 체크합니다.
- 승인된 데이터는 즉시 서버 API를 통해 DB에 저장됩니다.

---

## 4. 에이전트 팀 구성도

| 에이전트명 | 주요 역할 | 특이사항 |
| :--- | :--- | :--- |
| **🕵️ 뉴스 스카우터** | 주제 선정 및 비율 조절 | 엔터/스포츠 위주 선정 |
| **✍️ 콘텐츠 작가** | 영어 문장 10개 집필 | 난이도 Level 3 준수 |
| **👑 1타 강사** | 구문 분석 및 뉘앙스 해설 | 한자 사용 절대 금지 |
| **🛡️ 기술 QA** | JSON 형식 및 키워드 생성 | 기술적 무결성 담당 |
| **👨‍⚖️ 최종 감수관** | 9대 기준 최종 승인 | 반려 시 재작업 요청 |

---

## 5. 기대 효과
- **품질 유지**: 각 뉴스마다 깊이 있는 해설과 정확한 구문 분석이 보장됩니다.
- **안정성**: 대량의 텍스트 생성 시 발생할 수 있는 '끊김 현상'이 없습니다.
- **사용자 맞춤**: 사용자가 원하는 비율(정치 비중 낮춤)이 정확히 반영됩니다.
