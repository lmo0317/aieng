# Trend Eng 프로젝트 지침서 (Mandates)

이 문서는 AI 어시스턴트(Gemini)가 프로젝트를 수행할 때 반드시 준수해야 하는 최우선 지침입니다.

## 1. 페이지 명칭 정의 (Conversation & Documentation Rules)
모든 대화와 문서화 작업에서 다음 명칭을 사용합니다.

- **입력 페이지**: 루트 경로(`/`) 접속 시 나타나는 초기 화면. 학습 주제(텍스트/뉴스)를 입력하고 '학습 시작' 버튼이 있는 페이지.
- **학습 페이지**: '학습 시작' 버튼 클릭 후 생성된 10개의 문장을 순차적으로 공부하는 페이지 (해석, 문장 구조, AI 가이드 제공).
- **복습 페이지**: 내비게이션의 '복습하기' 또는 '학습 히스토리' 메뉴를 통해 접속하는 과거 학습 목록 페이지.

## 2. 프로젝트 핵심 컨셉
- **브랜드**: Trend Eng (트렌드 맞춤형 영어 학습)
- **핵심 가치**: 사용자 관심사(Trend)에 기반한 **맞춤형(Customized)** 영어 문장 및 학습 가이드 제공.

## 3. 기술적 제약 및 핵심 기능
- **AI 채팅 및 LLM 통일**: 
    - **모델**: 프로젝트의 모든 텍스트 분석 및 AI 채팅은 기본적으로 설정된 단일 Gemini 모델(`gemini-3.1-flash-lite-preview` 등)로 통일하여 사용합니다.
    - **음성/화상 제거**: 기존의 WebRTC 기반 실시간 음성/화상 채팅은 사용하지 않으며, 텍스트 전용 채팅으로 운영합니다.
- **품사 분석(POS Analysis) 제외**: 문장의 핵심 성분인 품사 분석 기능은 더 이상 사용하지 않으며, '문장 구성 요소(Sentence Structure)'와 'AI 학습 가이드(Explanation)'에 집중함.

## 4. 에이전트 및 스킬 관리 규칙 (Directory Rules)
- **Gemini CLI 전용 폴더 사용**: Gemini CLI는 프로젝트 내에서 사용하는 에이전트 지침서, 스킬 정의, 그리고 이를 실행하기 위한 스크립트들을 **반드시 `.gemini` 폴더 하위**에서만 관리하고 참조합니다.
- **스킬 중심 구조 (Skill-Centric)**: 모든 실행 스크립트(`.js`)는 루트의 `scripts/` 폴더가 아닌, 해당 기능이 정의된 `.gemini/skills/[skill-name]/scripts/` 하위에 위치해야 합니다.
- Single Source of Truth: 프롬프트 설계도(`.md`)와 실제 실행 스크립트는 항상 `.gemini` 안에서 한 쌍으로 관리되어 데이터 파편화를 방지합니다.
- **임시 파일 청소 (Auto Cleanup)**: 데이터 생성 및 업로드 등 모든 작업이 완료된 후에는 해당 작업 과정에서 생성된 모든 임시 파일(`.json`, 임시 `.js`, 로그 등)을 **반드시 즉시 삭제**하여 저장소를 항상 깨끗하게 유지해야 합니다.
- Claude 폴더 보호: `.claude` 폴더는 사용자가 Claude Code를 위해 별도로 만들어둔 영역이므로, Gemini는 이 폴더의 내용을 수정하거나 기준으로 삼지 않습니다.
