# 🚀 News Learning - Star Tutor Pipeline

## 🚨 CRITICAL: Failure Prevention Protocol (실수 방지 금기 사항)
당신은 과거에 다음과 같은 실수를 반복했습니다. **이 섹션을 읽는 즉시 다음 규칙을 뇌리에 각인하십시오.**

1.  **중복 체크 필수 (Strict Duplicate Check)**: 작업을 시작하기 전 **무조건** `https://minohlee.mooo.com/api/trends/saved`를 확인하여 이미 등록된 뉴스의 **제목, keywords, 핵심 인물/사건** 중 하나라도 겹치면 제외하십시오. 제목이 달라도 같은 사건을 다루면 중복입니다.
2.  **AI 내부 지식 의존 절대 금지**: 실시간 검색을 통해서만 내용을 작성하십시오.
3.  **최종 보고에 제목 포함**: 최종 보고 시 생성된 모든 기사의 제목을 명확히 리스트업 하십시오.
4.  **구체성 확보**: 기사에는 구체적인 날짜, 수치, 인물 발언, 사건 장소 등이 100% 실시간 사실에 기반하여 포함되어야 합니다. '날 것'의 소식을 전하십시오.
5.  **수량 준수 필수**: 사용자가 요청한 기사 개수(COUNT)를 반드시 준수하십시오. 출력 길이 제한을 핑계로 임의로 줄이는 것은 직무유기입니다.

## 📋 Mandatory Workflow (필수 워크플로우)

### Phase 1: Duplicate Defense (중복 철저 방지 - 시작 전 필수)
- **실시간 저장소 확인**: 작업을 시작하기 전, **반드시 `https://minohlee.mooo.com/api/trends/saved`를 호출**하여 기존 저장된 뉴스 전체 목록을 가져오십시오.
  - **윈도우(win32) 주의**: 터미널에서 `curl`을 직접 사용할 경우 파워쉘 별칭(Invoke-WebRequest) 문제로 멈출 수 있으니, 반드시 **`curl.exe`**를 사용하십시오.
- **중복 판정 기준 (3단계 필터 - 모두 통과해야 선정 가능)**:
  1. **제목 유사도**: 서버에 이미 존재하는 제목과 동일하거나 매우 유사한 뉴스는 제외합니다.
  2. **핵심 엔티티 중복**: 저장된 뉴스의 `keywords` 필드를 분석하여 동일한 **인물명, 기업명, 법안명, 사건명**이 핵심 주제로 등장하면 제외합니다. (예: 이미 "트럼프 관세" 뉴스가 있으면, 같은 관세 정책을 다른 각도에서 다룬 기사도 제외)
  3. **사건 동일성**: 제목과 키워드가 달라도 **동일한 핵심 사건**을 다루는 뉴스는 카테고리가 달라도 중복으로 간주하여 제외합니다.
- **블랙리스트 구성**: 위 필터를 적용한 뒤, 제외 대상 키워드/인물/사건 목록을 내부적으로 정리하고 이를 fetch-news.js 실행 후 후보 선별 시 반드시 참조하십시오.
- **새로운 주제 선정**: 블랙리스트에 없는 **완전히 새로운 소식**만 선별하여 다음 단계로 넘어갑니다. 후보가 부족하면 fetch-news.js를 추가로 실행하거나 다른 검색어로 재시도하십시오.

### Phase 2: News Scout (신규 주제 수집)
- **카테고리 파라미터 파싱**: 사용자 명령에서 카테고리를 추출하십시오.
  - `/news 정치 2개` → CATEGORY=정치, COUNT=2
  - `/news 테크 1개` → CATEGORY=테크, COUNT=1
  - `/news 엔터 2개` → CATEGORY=엔터, COUNT=2 (연애+스포츠 혼합)
  - `/news 2개` (카테고리 없음) → CATEGORY=전체, COUNT=2
- **카테고리별 fetch-news.js 실행**: 반드시 해당 카테고리 인자를 전달하여 실행하십시오.
  - `node .gemini/skills/news/scripts/fetch-news.js 정치`
  - `node .gemini/skills/news/scripts/fetch-news.js 테크`
  - `node .gemini/skills/news/scripts/fetch-news.js 엔터`
- **카테고리 고정 (DB 저장 규칙)**: 생성하는 기사의 `category` 필드는 아래 값 중 하나만 사용하십시오. **'엔터' 사용 절대 금지.**
  - 정치 요청 → `"정치"`
  - 테크 요청 → `"테크"`
  - 엔터 요청 → 스포츠 기사면 `"스포츠"`, 연예/엔터테인먼트 기사면 `"연애"`
- **실시간 팩트 체크 (Full Title 확보)**: 
  - 선정된 주제를 `google_web_search`로 검색하여 오늘 기준 최신 상세 내용을 직접 확인하십시오.
  - **중요**: 만약 `fetch-news.js`에서 가져온 제목이 `...`으로 끝나는 등 생략되어 있다면, 반드시 검색을 통해 **원문의 전체 제목(Full Title)**을 찾아 `news_title`로 사용하십시오. 축약된 제목을 그대로 사용하는 것은 금지입니다.


### Phase 2: Sequential Production (순차 생성 - 핵심)
- **중요**: 요청받은 개수의 기사를 하나씩 순차적으로 처리합니다. (1기사 완료 -> 서버 저장 -> 다음 기사 시작)
- 각 기사 생성 시 내부적으로 **1부터 10까지 문장 번호를 명시**하며 작성하십시오.
- **문장당 퀄리티**: AI 설명(`explanation`)은 반드시 **5문장 이상의 1타 강사 강의**여야 합니다.

### Phase 3: Star English Tutor (교육 내용)
- **자연스러운 번역**: 보도 어투의 세련된 한국어.
- **초정밀 구조 분석 (sentence_structure)**: 
    - 문장 성분(주어, 동사, 목적어, 보어, 수식어)은 **한글**로 표기하되, 해당 성분에 대응하는 **영어 단어는 원문 그대로** 유지합니다.
    - S, V, O 약어 사용을 지양하고 한글 용어를 사용합니다.
    - 1~5형식을 명시하고 각 성분의 역할을 설명합니다.
    - 예시: `[주어: The company] + [동사: launched] + [목적어: a new product] (3형식 문장)`
    - **주의**: 분석 내용에서 영어 단어를 한국어로 번역하여 적지 마십시오. 오직 문장 성분명만 한글입니다.
- **소름 돋는 AI 가이드**: 뉴스 맥락, 문법 포인트, 뉘앙스 차이, 실전 활용 팁, 격려 포함 (최소 5문장).
- **핵심 어휘 (`voca`)**: 문장당 3-5개 필수. 형식: `단어(품사): 뜻`.

### Phase 4: Quiz Maker (어휘 강화형 퀴즈)
- **어휘 중심**: 기사 내용 확인이 아닌, 배운 **핵심 어휘(`voca`)의 뜻과 쓰임**을 복습하는 형태여야 합니다.
- 기사당 **10개 모두 4지 선다 객관식(multiple_choice)**으로 생성합니다. (빈칸 채우기 절대 금지)
- 질문은 반드시 **한글**로 작성하며, 선택지는 반드시 4개여야 합니다.

### Phase 5: Server Sync (운영 서버 저장 - 안전 프로토콜)
- **환경 점검**: 실행 전 `.env` 파일의 `SERVER_URL`이 운영 서버(`https://minohlee.mooo.com`)를 가리키고 있는지 반드시 확인하십시오. 만약 `localhost`로 되어 있다면 운영 서버 주소를 우선적으로 사용하여 전송해야 합니다.
- **안전 전송 (Windows Safe)**: 윈도우 환경의 인코딩 및 명령어 길이 제한 오류를 방지하기 위해, **절대 긴 JSON 데이터를 명령어 인자로 직접 전달하지 마십시오.** 
    1. 반드시 데이터를 임시 JSON 파일(예: `temp_news_data.json`)로 먼저 작성하십시오.
    2. 해당 파일 경로를 인자로 하여 `save-to-server.js`를 실행하십시오.
    3. 작업 완료 후 임시 파일을 반드시 삭제하십시오.
- **최종 검증**: 스크립트 출력 결과에서 `✅ Server Save SUCCESS!` 또는 `success: true` 메시지를 실제로 확인한 경우에만 성공으로 보고하십시오.

## 🛠️ JSON Data Structure (데이터 규격)
모든 데이터는 아래 구조를 엄격히 준수하여 JSON으로 생성해야 합니다.

```json
{
  "content": [
    {
      "news_title": "뉴스 제목 (한글)",
      "category": "경제",
      "sentences": [
        {
          "english": "Sentence...",
          "korean": "번역...",
          "sentence_structure": "[주어: ...] + [동사: ...] + [목적어: ...] (N형식 문장)",
          "explanation": "5문장 이상의 상세 강의...",
          "voca": "단어(품사): 뜻"
        }
      ],
      "quiz": [
        {
          "type": "multiple_choice",
          "question": "어휘 관련 질문...",
          "options": ["정답", "오답1", "오답2", "오답3"],
          "answer": "정답"
        }
      ]
    }
  ]
}
```

## 🛡️ Guardrails (행동 수칙)
1. **한자 절대 금지**: 모든 텍스트에서 한자를 제거하십시오.
2. **구조 분석 용어만 한글화**: `sentence_structure` 내 분석 용어(주어, 동사 등)만 한글로 표기하고, **실제 영어 단어는 절대 번역하지 마십시오.** (예: `[주어: The stock market]` - O, `[주어: 주식 시장이]` - X)
3. **어휘 중심 퀴즈**: 본문 내용이 아닌 voca 기반의 어휘 복습형으로 설계.
4. **객관식 전용**: 4지 선다 객관식만 허용하며 빈칸 채우기는 사용하지 않음.
5. **실시간 팩트**: 뻔한 정보가 아닌 오늘 자 실시간 검색 기반의 구체적 정보 포함.
