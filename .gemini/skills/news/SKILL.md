# 🚀 News Learning JSON Inspector

이 스킬은 `output/` 폴더 내에 새롭게 생성되거나 추가된 뉴스 학습 JSON 파일들의 품질과 형식을 엄격하게 검수하는 전용 도구입니다. 기존 데이터 수집 및 생성 기능은 제거되었습니다.

## 🛠️ 주요 명령어
1.  **파일 검수**: `/news 검수` (또는 `/news 확인해줘`, `/news 리뷰해줘` 등)
    *   **워크플로우**: 
        1. `output/` 폴더에 있는 최신 JSON 파일(예: `news_guide_*.json` 등)을 찾아서 읽습니다.
        2. 아래의 **1타 강사 검수 기준**에 따라 JSON 파일의 내용을 꼼꼼하게 평가합니다.
        3. 검수 결과(합격/불합격 및 수정 필요 사항)를 사용자에게 리포트합니다.

## 📋 1타 강사 검수 기준 (Auditor's Checklist)
JSON 파일 내의 각 학습 주제(객체)에 대해 다음 항목들을 확인하세요:

1. **[구조 및 개수]**: 
    - 파일이 올바른 JSON 배열 또는 객체 형식인가?
    - 각 주제별로 정확히 **10개**의 문장(`sentences`)과 **10개**의 퀴즈(`quiz`)가 존재하는가?
2. **[형식 및 언어]**: 
    - `explanation`과 `sentence_structure` 필드가 존재하며, 설명 및 명칭이 **한글**로 상세히 작성되었는가? (예: S(주어: ~) + V(동사: ~))
3. **[품질]**: 
    - 각 문장에 대한 `explanation`이 단순 해석에 그치지 않고, 문법/시제/단어의 뉘앙스 차이 등 **최소 3문장 이상의 깊이 있는 1타 강사 스타일의 설명**을 담고 있는가?
    - 실전 비즈니스나 토익 등에서의 활용 팁이 포함되어 있는가?
4. **[퀴즈 유효성]**:
    - 퀴즈의 정답(`answer`)이 제공된 보기(`options`) 중에 존재하는가?
    - 퀴즈의 해설(`explanation`)이 명확한가?

## 📂 실행 환경
- **주요 작업 대상 폴더**: `output/` (예: `output/news_guide_*.json`)
- 에이전트는 파일 시스템 도구(예: `read_file`, `list_directory`)를 사용하여 `output/` 폴더의 JSON 파일을 직접 읽고 위 기준에 따라 분석을 수행합니다. 별도의 생성 스크립트를 실행하지 않습니다.
