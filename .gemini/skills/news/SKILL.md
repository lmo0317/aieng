# 🚀 News Learning - Star Tutor Pipeline

## 🚨 CRITICAL: Failure Prevention Protocol (실수 방지 금기 사항)
당신은 과거에 다음과 같은 실수를 반복했습니다. **이 섹션을 읽는 즉시 다음 규칙을 뇌리에 각인하십시오.**

1.  **AI 내부 지식 의존 절대 금지**: 당신이 알고 있는 '상식'이나 '학습 데이터'로 기사를 쓰지 마십시오. (예: 뻔한 구글 업데이트, 예전 소식 등)
2.  **실시간 검색 필수**: 뉴스 제목을 가져온 후, **반드시 `google_web_search` 도구를 사용**하여 해당 사건의 **오늘(2026-03-22) 기준 상세 내용**을 직접 확인한 후 기사를 작성하십시오.
3.  **중복 뉴스 생성 금지 (Duplicate Check)**: 작업을 시작하기 전, **반드시 `http://aieng.cafe24app.com/api/trends/saved`를 확인**하여 이미 오늘 날짜로 등록된 뉴스 제목들과 중복되지 않는 새로운 주제를 선정하십시오. 주제가 비슷하더라도 핵심 사건이 겹치면 제외하십시오.
4.  **구체성 확보**: 기사에는 구체적인 날짜, 수치, 인물 발언, 사건 장소 등이 100% 실시간 사실에 기반하여 포함되어야 합니다. '날 것'의 소식을 전하십시오.
5.  **수량 준수 필수**: 사용자가 요청한 기사 개수(COUNT)를 반드시 준수하십시오. 출력 길이 제한을 핑계로 임의로 줄이는 것은 직무유기입니다.

## 📋 Mandatory Workflow (필수 워크플로우)

### Phase 1: News Scout (주제 수집 & 중복 체크)
- **중복 확인**: `http://aieng.cafe24app.com/api/trends/saved` API를 호출하여 오늘 생성된 뉴스 목록을 가져옵니다.
- **주제 선정**: `fetch-news.js` 결과 중, **서버에 이미 존재하는 제목이나 주제와 겹치지 않는 것**을 우선순위로 선정합니다.
- **실시간 팩트 체크**: 선정된 주제를 `google_web_search`로 검색하여 기사 내용을 완벽히 파악하십시오.
- 뉴스 제목은 반드시 **100% 한글**로 번역합니다.
- **카테고리(category) 한글화**: `category` 필드(태그)는 반드시 **한글**로 작성합니다. (예: "경제", "테크", "스포츠", "연애", "정치")

### Phase 2: Sequential Production (순차 생성 - 핵심)
- **중요**: 요청받은 개수의 기사를 하나씩 순차적으로 처리합니다. (1기사 완료 -> 서버 저장 -> 다음 기사 시작)
- 각 기사 생성 시 내부적으로 **1부터 10까지 문장 번호를 명시**하며 작성하십시오.
- **문장당 퀄리티**: AI 설명(`explanation`)은 반드시 **5문장 이상의 1타 강사 강의**여야 합니다.

### Phase 3: Star English Tutor (교육 내용)
- **자연스러운 번역**: 보도 어투의 세련된 한국어.
- **초정밀 구조 분석 (sentence_structure)**: 
    - 문장 성분(주어, 동사, 목적어, 보어, 수식어)은 **한글**로 표기하되, 해당 성분에 대응하는 **영어 단어는 원문 그대로** 유지합니다.
    - S, V, O 약어 사용을 지양하고 한글 용어를 사용합니다.
    - 1~5형식을 명시하고 각 성분의 역할을 설명합니다.
    - 예시: `[주어: The company] + [동사: launched] + [목적어: a new product] (3형식 문장)`
    - **주의**: 분석 내용에서 영어 단어를 한국어로 번역하여 적지 마십시오. 오직 문장 성분명만 한글입니다.
- **소름 돋는 AI 가이드**: 뉴스 맥락, 문법 포인트, 뉘앙스 차이, 실전 활용 팁, 격려 포함 (최소 5문장).
- **핵심 어휘 (`voca`)**: 문장당 3-5개 필수. 형식: `단어(품사): 뜻`.

### Phase 4: Quiz Maker (어휘 강화형 퀴즈)
- **어휘 중심**: 기사 내용 확인이 아닌, 배운 **핵심 어휘(`voca`)의 뜻과 쓰임**을 복습하는 형태여야 합니다.
- 기사당 **10개 모두 4지 선다 객관식(multiple_choice)**으로 생성합니다. (빈칸 채우기 절대 금지)
- 질문은 반드시 **한글**로 작성하며, 선택지는 반드시 4개여야 합니다.

### Phase 5: Server Sync (운영 서버 저장 - 안전 프로토콜)
- **환경 점검**: 실행 전 `.env` 파일의 `SERVER_URL`이 운영 서버(`http://aieng.cafe24app.com`)를 가리키고 있는지 반드시 확인하십시오. 만약 `localhost`로 되어 있다면 운영 서버 주소를 우선적으로 사용하여 전송해야 합니다.
- **안전 전송 (Windows Safe)**: 윈도우 환경의 인코딩 및 명령어 길이 제한 오류를 방지하기 위해, **절대 긴 JSON 데이터를 명령어 인자로 직접 전달하지 마십시오.** 
    1. 반드시 데이터를 임시 JSON 파일(예: `temp_news_data.json`)로 먼저 작성하십시오.
    2. 해당 파일 경로를 인자로 하여 `save-to-server.js`를 실행하십시오.
    3. 작업 완료 후 임시 파일을 반드시 삭제하십시오.
- **최종 검증**: 스크립트 출력 결과에서 `✅ Server Save SUCCESS!` 또는 `success: true` 메시지를 실제로 확인한 경우에만 성공으로 보고하십시오.

### Phase 6: Telegram Notification (최종 보고)
- 모든 작업 완료 후 [생성된 기사 제목 리스트], [주제별 개수], [총 문장 수], [AI 학습 가이드 반영 여부]를 포함하여 정중하게 보고합니다.
- **서버 반영 확인**: 보고서에 반드시 "운영 서버(aieng.cafe24app.com) 반영 완료" 문구를 포함하십시오.

## 🛠️ JSON Data Structure (데이터 규격)
모든 데이터는 아래 구조를 엄격히 준수하여 JSON으로 생성해야 합니다.

```json
{
  "content": [
    {
      "news_title": "뉴스 제목 (한글)",
      "category": "경제",
      "sentences": [
        {
          "english": "Sentence...",
          "korean": "번역...",
          "sentence_structure": "[주어: ...] + [동사: ...] + [목적어: ...] (N형식 문장)",
          "explanation": "5문장 이상의 상세 강의...",
          "voca": "단어(품사): 뜻"
        }
      ],
      "quiz": [
        {
          "type": "multiple_choice",
          "question": "어휘 관련 질문...",
          "options": ["정답", "오답1", "오답2", "오답3"],
          "answer": "정답"
        }
      ]
    }
  ]
}
```

## 🛡️ Guardrails (행동 수칙)
1. **한자 절대 금지**: 모든 텍스트에서 한자를 제거하십시오.
2. **구조 분석 용어만 한글화**: `sentence_structure` 내 분석 용어(주어, 동사 등)만 한글로 표기하고, **실제 영어 단어는 절대 번역하지 마십시오.** (예: `[주어: The stock market]` - O, `[주어: 주식 시장이]` - X)
3. **어휘 중심 퀴즈**: 본문 내용이 아닌 voca 기반의 어휘 복습형으로 설계.
4. **객관식 전용**: 4지 선다 객관식만 허용하며 빈칸 채우기는 사용하지 않음.
5. **실시간 팩트**: 뻔한 정보가 아닌 오늘 자 실시간 검색 기반의 구체적 정보 포함.
