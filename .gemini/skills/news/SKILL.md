# 🎓 News Learning (High-Fidelity Pipeline)

이 스킬은 데이터 유실과 품질 저하를 막기 위해 엄격한 순차적 파이프라인을 가동합니다.

## 💡 주요 명령어
1.  **데이터 삭제**: `/news delete`
    *   오늘 날짜의 뉴스 데이터를 DB에서 깨끗하게 지웁니다. (새로운 작업을 시작할 때 권장)
2.  **데이터 생성**: `/news [개수]개 만들어줘`
    *   요청한 개수만큼 뉴스를 수집하고, 하나씩 정성 들여 가공하여 서버에 누적 저장합니다.

## 👥 에이전트 파이프라인 (The Factory)

### [Step 1: Scout ALL]
*   **🕵️ 뉴스 스카우터**: **게임, 엔터테인먼트, 스포츠** 테마에서 중복 없이 요청받은 수만큼 주제를 먼저 선정합니다.
*   **지침**: 제목은 반드시 **100% 한국어**로만 작성하며, 영어나 한자가 섞이지 않도록 합니다.

### [Step 2: Sequential Production]
*에이전트 팀은 리스트된 주제를 하나씩(One by One) 처리하여 출력 제한과 품질 저하를 방지합니다.*

1.  **✍️ 콘텐츠 작가**: 선정된 주제당 **정확히 10개**의 수준 높은 Level 3 영어 문장을 작성합니다.
    *   **주요 어휘**: 각 문장마다 핵심 단어 2~3개를 선정하여 `["word : 뜻", "word : 뜻"]` 형식으로 포함합니다. 반드시 **영어 단어와 한글 뜻**을 함께 적어야 합니다.
2.  **👑 1타 강사**: 
    *   **구문 분석**: "주어(Word) + 동사(Word)" 형태로 실제 단어를 매칭하여 친절하게 분석합니다.
    *   **생활 영어**: 시험용 설명 대신 원어민 뉘앙스와 실생활 활용 팁을 4~5문장으로 상세히 작성합니다. (이모지 최소화)
3.  **🛡️ 기술 QA**: JSON 무결성 확보 및 해당 뉴스의 핵심 **한글 키워드 5개**(#태그)를 생성합니다.

### [Step 3: Supreme Audit & Sync]
*   **👨‍⚖️ 최종 감수관**: 아래 9대 기준을 검수하여 승인된 데이터만 서버에 즉시 전송합니다.
*   **누적 저장**: 서버의 `/api/trends/save`를 호출하여 기존 데이터를 지우지 않고 차곡차곡 쌓습니다.

## 🚫 9대 필수 감수 기준 (Auditor's Checklist)
1.  **[문장 개수 완비]**: 각 주제별로 정확히 **10개**의 문장이 들어있는가?
2.  **[제목 100% 한글]**: `news_title`에 영어 캐릭터가 단 하나라도 포함되었는가? (무조건 한글화)
3.  **[주제 흥미도]**: 게임, 엔터, 스포츠 테마인가? (정치/전쟁/경제 거시 뉴스는 반려)
4.  **[한자 금지]**: 한국어 텍스트에 한자가 섞여 있는가?
5.  **[구문 분석 구체성]**: `S+V+O` 식이 아닌, 단어 매칭형 분석인가?
6.  **[생활 영어 중심]**: 실생활 활용 중심의 해설인가?
7.  **[키워드 완비]**: 한글 해시태그 5개가 포함되었는가?
8.  **[어휘 형식 준수]**: `voca` 배열에 `단어 : 뜻` 형식으로 영어와 한글이 모두 포함되었는가? (영어만 있으면 반려)
9.  **[기술 규격]**: JSON 필드명과 이스케이프가 완벽한가?
